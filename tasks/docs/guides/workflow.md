# タスク管理ワークフロー

## 1. 新規タスク・プロジェクトの追加

1. `inbox/`ディレクトリに新規アイテムを追加
2. 定期的なトリアージで以下を判断：
   - プロジェクトかタスクか
   - 実行者（人間/AI）の判断
   - 優先度の設定

## 2. プロジェクトのブレイクダウン

プロジェクトは必要に応じて以下の形でブレイクダウン可能：
- 新規プロジェクトとして分離
- 具体的なタスクへの分解
- 実行可能な単位への細分化

## 3. タスクの実行

- 人間による実行：通常のタスク実行フロー
- AIによる実行：自動化システムによる実行
  - GitHub Issues自動収集
  - Scrapboxからのタスク抽出
  - 多言語翻訳など

## 4. 多言語コンテンツ管理

1. 原文の作成・確認
2. 翻訳言語の選定
3. 翻訳実行（人間/AI）
4. レビュー・フィードバック
5. 公開・展開
